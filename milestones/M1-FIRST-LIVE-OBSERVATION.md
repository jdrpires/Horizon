# Milestone M1: First Live Observation

Status: Recorded

## Context

Milestone M1 records the first moment in which Horizon observed a real physical Asset.

The field test was performed with:

- Asset: Citroën C3
- Device: ELM327 Bluetooth adapter
- Runtime device: Android/Realme
- Bridge: Android OBD Bridge

This was not a simulation. The readings came from a physical vehicle through an ELM327 adapter connected to the Android OBD Bridge.

## Observed Result

The first live observation produced the following readings:

- RPM: `805 rpm`
- Engine temperature: `76 °C`
- Voltage: `12.31 V`
- Status: `leitura concluída`
- Timestamp: generated by the Android OBD Bridge during the reading

## Architectural Meaning

The test confirmed the intended boundary:

- Bluetooth remained outside Horizon Core.
- Android acted as the Bridge.
- Horizon preserved its architectural boundary.
- Horizon Core continues to receive only canonical observations.

The physical integration did not require Horizon Core to know Bluetooth, ELM327, Android, RFCOMM, OBD, or vehicle-specific transport details.

## Learnings

- The Android Bridge strategy is viable.
- The ELM327 adapter responded to the basic OBD-II PIDs.
- The parser produced usable values for RPM, engine temperature, and voltage.
- The next step is to send the JSON payload to Horizon through a Gateway/API.
