# Horizon Roadmap

Status: Prepared for Chief Software Architect review

## Purpose

This roadmap separates completed Generation 1 work from planned Generation 2 and future Generation 3 vision.

It is directional. It does not authorize implementation by itself. New capabilities still require the official Horizon governance path: Vision, Workshop, RFC, ADR, SPEC, Capability, Sprint, Demo, and Merge.

## Generation 1: Foundation Complete

Status: Completed in Alpha 1.0

Generation 1 established the memory-first foundation:

- Constitution, Manifesto, Engineering Playbook, and governance.
- Horizon Kernel.
- Horizon Event Platform.
- Horizon Protocol.
- Asset Domain.
- Observation Domain.
- Application Layer.
- Horizon Lab.
- Timeline and Replay.
- Current State.
- Local JSON Storage.
- Experience Layer.
- Observation Catalog.
- Collector Framework.
- Android/ELM327 OBD Collector Spike.
- Android OBD Bridge.
- Milestone M1: First Live Observation.
- Live Ingestion Gateway.
- Horizon Mobile.

Generation 1 proves that Horizon can receive facts from a physical Asset without compromising Core boundaries.

## Generation 2: Intelligence From Memory

Status: Planned

Generation 2 should introduce intelligence only through memory and evidence.

Planned capabilities:

- Observation Value Model.
- Memory Engine.
- Knowledge Engine.
- Health Engine.
- Recommendation Engine.
- Conversation Engine.
- Living Digital Twin Runtime.
- Agent Framework.
- Natural Language Layer.

Generation 2 must preserve the principle that AI does not know the Asset. The Twin knows the Asset through identity, memory, context, history, and explainable knowledge.

## Generation 3: Operational Horizon

Status: Future vision

Generation 3 may focus on broader operational maturity after the Twin and knowledge path are established.

Potential areas:

- Production API boundary.
- Authentication and authorization.
- Multi-tenant runtime.
- Offline queue and background synchronization.
- Device and collector management.
- Deployment hardening.
- Web dashboard if it serves understanding rather than display.
- Long-term migration and compatibility policy.
- LTS support model.

Generation 3 must not turn Horizon into a generic telemetry platform. Operational maturity must serve Living Digital Twin understanding.

## Roadmap Rule

The roadmap is not permission to skip architecture.

Every meaningful item still requires its own documents, review, and evidence before implementation.
